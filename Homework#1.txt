

1.	Version control systems are a category of software tools that help a software team manage changes to source code over time.


2.	Repository is usually used to organize a single project. Repositories can contain folders and files, images and videos, spreadsheets and datasets etc.

3.	The main purpose of a repository is to store a set of files, as well as the history of changes made to those files.

4.	Git is a version control system (VCS) for tracking changes in computer files and coordinating work on those files among multiple people. It is primarily used for software development. This is the version control software on which GitHub is built on.

5.	GitHub is a code hosting platform version control and collaboration. It lets you and others to work together on projects from anywhere. This is kind of a storage location. 

6.	Git is a revision control system, a tool to manage your source code history. GitHub is a hosting service for Git repositories. So they are not the same thing: Git the tool, GitHub the service for projects that use Git. 

7.	Pull – this is downloading the partial updates. This helps us to fetch from and integrate with other repository or a local branch
	Push – Update remote references along with associated objects.
	Commit – record changes made to   the repository.

8.	Merge conflicts may occur if competing changes are made to the same line of a file or when a file is deleted that another person is attempting to edit.
